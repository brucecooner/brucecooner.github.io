Earth Slicer

Python

This project grew from my frustrations trying to make topographical models using the previous project (FLT to Topo). A top down topograpical model can require a lot of individual pieces, which have to be individually tracked and correctly placed.

So I turned the problem on its side and instead of tracing isolines, I traced elevations along vertical "slices" of earth, which just get placed next to each other to create the model. This reduced the placement/registration to a one dimensional problem.

For this project I switched from C# to Python, but the biggest functional change was using the USGS point height query service to get elevations instead of downloading large datasets, which greatly simplified the workflow.
I quickly found however that querying the elevations serially was too slow, so I refactored the query code to issue parallel queries. Another optimization was to cache returned elevations in a local text file, that way running the same jobs again did not even need to query the server.
This app was written to output the slices in svg files ready to load and use in the laser cutter without requiring any further workflow steps.
This has made for some interesting physical models, but I'm still thinking about how to use these results in an artistic way. They would make a good basis for a solid sculpture, giving you a ready made base for clay or other material.

FLT to topo

C#

After acquiring a laser cutter years ago, I wanted to make topographically inspired maps. For this to work I needed contour maps with the isolines at arbitrary elevations that weren't necessarily powers of ten.
I found out that I could download FLT/HDR format height maps from the USGS, so I wrote an application that could produce my arbitrarily contoured from these source files. I enjoyed breaking the problem down into smaller chunks of functionality, but my favorite part of this experience was using parallelism to massively decrease the time required to create the output data.
While this program did what I intended it to do, unfortunately I found working contour maps in physical form to be unwieldy due to the large number of individual pieces the process produced. Also, being forced to work with downloaded data which was broken on one degree boundaries quickly became limiting. So I abandoned this project, and started the Earth Slicer project to make working with topographical data easiser.

mandala creator

HTML/Javascript

I had so much fun playing with the spokes maker that I created a drawing app completely focused on radial symmetry.
With it you draw lines (straight or freeform), circles, or curves onto a bunch of radial slices of a canvas, and you can increase the number of to draw a whole lot of elements in a hurry.
I had more things in mind for this project but I got distracted and set it aside. It still wound up as something that's a lot of fun to play with though.
It's purely javascript, running in the browser, making extensive use of vector graphics.
Aspects of this project I particularly enjoyed were using object oriented design and classes to support the different drawing modes through a common interface, and figuring out how to create the snap-to system.

parametric circles

HTML/Javascript

I was pretty obsessed with radial symmetry for a while, and made yet another toy along that theme.
This one however focused on laying out a bunch of circles along a set of spokes, with parametric controls to change the layout dynamically. Again, I had more ideas for this project but got distracted. I was planning to expand the number of element types beyond just circles, and it obviously needed a lot more instruction and feedback for ease of use. This is another javascript project that runs in the browser. I made sure this project can export to svg format. One unexpectedly fun aspect of this one was creating a color selection wheel for changing the stroke/fill colors. It was not easy to accomplish but a lot of fun. Being parametric in nature, I was also able to add save/restore feature through local browser storage, which is pretty neat.
I did actually use this tool to export some circular art that went into a physical product made on my laser cutter.

Glowforge dashboard job scraper

Javascript

If you've owned a Glowforge laser cutter for any length of time, you know that the dashboard where you select a print job will eventually become a huge list dozens of pages long.
For the longest time, the dashboard offered no way to search this list, which led to endless scrolling to find anything you'd printed more than a few weeks before.
Each of these print jobs has a unique url, meaning you can visit them directly if you have the address, which gave me an idea...

My reponse was to create a javascript module that you add to the dashboard page (via the browser debug console) which hooks the scroll event and scrapes jobs and their urls as you scroll the page.
After scrolling through your job list, you return to the debugger and call a function in the module to list all of the jobs it has discovered in a tabular form, that you can drop directly into a spreadsheet.
I will regularly scrape all of the jobs in my dashboard and drop it into a sheet so Ihave a full and fully visible searchable list of all my print jobs.
Glowforge recently added a search bar to the dashboard, which helps, but it's still useful to have a single page view of my jobs list at times.

Woodworking

I've been a hobbyist level woodworker since I was young enough to wield a pull saw.
It's a skill that has served me well, letting me create a lot of interesting and useful things.


Many years ago a coworker wanted a driving simulator cockpit that exactly matched the dimensions of his BMW's driver's seat, to which he could mount a very expensive steering wheel controller he had recently acquired, and he asked me to help make it a reality. The constraints made this a challenging and interesting project, and the result was a beautiful piece of functional entertainment focused furniture which, as far as I know, still resides in the middle of his living room.


My wife requested a small three level shelf for her office space, and I took the opportunity to use a design I came up with some time ago that I call the "K Leg", where the front legs become an X with the cross pushed back, so that viewed from the side they form the shape of the letter K. I love the cantilever effect this lent to the middle shelf, and it was very practical on this design as it makes the center shelf easier to use.
